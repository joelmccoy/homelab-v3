# 0015 — Postgres: CloudNativePG

- **Status:** Accepted
- **Date:** 2026-05-03

## Context

Keycloak (and likely future apps) need Postgres. Options were the embedded H2 (Keycloak), Bitnami Postgres helm chart, or a dedicated operator. We want backups, HA-readiness, and per-app DBs in their own namespaces.

## Decision

Run CloudNativePG (CNPG) operator cluster-wide in `cnpg-system`. Each app declares its own `Cluster` CR in its own namespace (`Cluster` lives in the same ns as its workload). Storage backed by Longhorn.

## Consequences

- Per-app DB isolation by namespace; no central "shared Postgres" to migrate later.
- Native primitives for backups, restores, point-in-time recovery, replicas.
- App user/password Secrets auto-generated by CNPG; consumers reference them directly (no manual sealing for connection creds).
- Adds an operator (~256 MB RAM) but the value is multi-app — this is the only one we'll need.
